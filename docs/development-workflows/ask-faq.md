# docs/development-workflows/ask-faq.md

## 指示

{{ user_input }} の質問に対して、適切なドキュメントを参照してFAQ形式で回答してください

## 文脈情報

このドキュメントは、プロジェクトに関するFAQ（よくある質問）に対応するためのガイドラインです。AIエージェントは、ユーザからのFAQのような質問に対して、プロジェクト内のドキュメントを情報源として正確な回答を提供します。

## ルール

- プロジェクトの情報源として、**mdファイルのみ**を参照すること
- CLAUDE.mdを中心としたドキュメント構造を理解して回答すること
- 回答は正確で、参照元を明記すること
- 情報が見つからない場合は、素直に「分からない」と回答すること

## 手順

1. 以下の優先順位でドキュメントを**事前に読み込む**：
   - `/CLAUDE.md`: プロジェクト全体の設定
   - `**/CLAUDE.md`: その場所に限定した詳細情報
   - `docs/**/*.md`: 業務知識や暗黙知
2. {{ user_input }} の質問内容を分析し、読み込んだドキュメントから関連情報を抽出する
3. `*.md` ファイルから情報が見つからない場合、フォールバック戦略として`src/**/*.tsx`と`src/**/*.ts`のコメントからFAQを作成する
4. 出力形式に従って回答を作成する
5. 参照元を明記して情報の信頼性を担保する

## 出力形式

- マークダウン形式で出力してください
- 以下の構成で回答してください：
  - ## 回答
  - ## 参照元ドキュメント
  - ## 関連情報（必要に応じて）

## 入出力例 (Few-shots)

**例1:**
入力:
```
このプロジェクトのコーディングルールはどこに書かれていますか？
```

出力:
```
## 回答

このプロジェクトのコーディングルールは以下の場所に記載されています：

- **docs/coding-rules/CLAUDE.md**: コーディングルールの詳細
- **各ディレクトリの**/CLAUDE.md**: その場所に限定したコーディング情報

## 参照元ドキュメント

- docs/coding-rules/CLAUDE.md
- /CLAUDE.md

## 関連情報

コードを作成・変更する際は、docs/coding-rules/*.md を読み込んでから実装することが推奨されています。
```

**例2:**
入力:
```
開発サーバーの起動方法を教えてください
```

出力:
```
## 回答

開発サーバーの起動は以下のコマンドで行います：

\`\`\`bash
pnpm dev
\`\`\`

## 参照元ドキュメント

- /README.md (Common Commands セクション)

## 関連情報

その他の利用可能なコマンド：
- `pnpm build`: プロダクション用ビルド
- `pnpm start`: プロダクションサーバー起動
- `pnpm lint`: Biomeリンティング
- `pnpm test`: Vitestテスト実行
```

## 注意点

- 情報が見つからない場合は、推測せずに「分からない」と回答してください
- 参照元ドキュメントは必ず明記してください
- mdファイル以外のソースコードを直接参照する場合は、その旨を明記してください
