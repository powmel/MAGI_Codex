# docs/development-workflows/create-doc.md

## 指示

{{ user_input }} を元に、docs/ 配下の該当する場所にドキュメントを作成してください

## 文脈情報

このドキュメントは、docs/ 配下の適切な場所にドキュメントを作成するためのガイドラインです。AIエージェントは、ユーザからの要求に基づいて、プロジェクトのドキュメント構造に従った適切なドキュメントを作成します。

## ルール

- プロジェクトのドキュメント構造を理解して適切な場所にドキュメントを作成すること
- 既存のCLAUDE.mdファイルの情報を活用すること
- ドキュメントの内容は正確で、参照元を明記すること
- 情報が不足している場合は、素直に「分からない」と回答すること

## 手順

1. 以下の優先順位でドキュメントを**事前に読み込む**：
   - `/CLAUDE.md`: プロジェクト全体の設定
   - `**/CLAUDE.md`: その場所に限定した詳細情報
   - `docs/**/*.md`: 業務知識や暗黙知
2. {{ user_input }} の内容を分析し、以下を判断する：
   - ドキュメントの種類（コーディングルール/ワークフロー/ドメイン知識）
   - 対象となるレイヤーやシーン
   - 関連する既存ドキュメントの有無
3. 既存ドキュメントがある場合、「update-docs を使用してドキュメントを更新してください」と出力して、手順を終了してください
4. 適切な docs ディレクトリを決定する：
   - `coding-rules/`: プログラムの書き方、レイヤー別実装ルール
   - `development-workflows/`: Git操作、レビュー、リリース等のワークフロー
   - `domain-knowledges/`: 仕様書、用語集、業務ルール
5. `development-workflows/` の場合、「create-development-workflow を使用してワークフローを作成してください」と出力して、手順を終了してください
6. {{ user_input }} を要約して、ドキュメント名 {{ doc_name }} を考えてください
7. タイトルのドキュメント名と docs/{{ directory }}/{{ doc_name }}.md のファイル名が良さそうか、（破壊的変更・セキュリティ影響・課金発生がある場合を除き）確認せずに続行してください
8. 出力形式に合わせて、ドキュメントを作成してください

## 出力形式

- マークダウン形式で出力してください
- frontmatterの記述を忘れないこと

## 注意点

- ドキュメント作成時は必ず既存のCLAUDE.mdファイルを参照すること
- 分からない場合は推測せず「分からない」と回答すること
