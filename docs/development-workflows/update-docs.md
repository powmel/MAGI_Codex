# docs/development-workflows/update-docs.md

## 指示

{{ user_input }} を元に、該当する docs/ 配下の複数のドキュメントを更新してください

## 文脈情報

このドキュメントは、docs/ 配下の複数の既存ドキュメントを更新するためのガイドラインです。

## ルール

- 対話を元に、該当する docs/ 配下の複数のドキュメントを更新してください
- 既存ドキュメントの構造と形式を維持すること
- frontmatterやコメントブロックなどの既存の書式を保持すること

## 手順

1. docs/ 配下の複数のドキュメントを更新する場合、
2. 更新対象は、docs/ 配下の md ファイル全てで、その中から、該当するドキュメントを特定する
3. 対話して、{{ user_input }} を取得する
4. {{ user_input }} を元に、複数ある docs/ 配下のドキュメントのうち、どのドキュメントを更新するかを決定する
5. 更新対象のドキュメントの内容を読み込み、現在の構造と形式を把握する
6. {{ user_input }} の内容に基づいて、適切な更新内容を決定する
7. 更新対象のドキュメントのどの部分をどのように更新するか、（破壊的変更・セキュリティ影響・課金発生がある場合を除き）確認せずに続行する
8. そのまま更新対象のドキュメントを更新する

## 出力形式

- マークダウン形式で出力してください

## 入力データ

更新したいドキュメントのファイルパスまたは内容、および更新内容の詳細

## 入出力例 (Few-shots)

例1:
入力:
```
docs/coding-rules/react-components.md に新しいhookのルールを追加したい。
「カスタムhookは必ず hooks ファイルに分離すること」
```

出力:
```
更新対象: docs/coding-rules/react-components.md
更新内容:
- ## ルール セクションに「カスタムhookは必ず hooks ファイルに分離すること」を追加

この方針で更新を実行します。
```

## 注意点

- 更新対象は、docs/ 配下の md ファイル全てです
- 分からない場合は、分からないとしてください
